BrainCache — Frontend Architecture

How React renders · Routing · Components · Hooks · Data Flow

① Entry Point main.jsx → App.jsx — this is where everything starts
main.jsx
The very first file Node runs.
ReactDOM.createRoot() mounts the entire app into
the #root div in index.html.

Wraps everything in <React.StrictMode>.
Imports global.css here (only place).
App.jsx — Root Component (no hooks used here)
Two jobs only: wrap providers and declare routes. No state, no effects.
<AuthProvider> ← wraps EVERYTHING so all children can access auth state <BrowserRouter> ← enables React Router (URL watching) <AppRoutes> ← inner component that reads useAuth() and renders routes <Routes> ├─ <PublicRoute> → /login, /register └─ <PrivateRoute> → / · /archive · /profile
Why AppRoutes is separate?
useAuth() hook can only be called inside AuthProvider's children.

App.jsx renders AuthProvider, so AppRoutes (a child) can safely call useAuth() to decide which routes are visible.
AuthProvider wraps → context value flows down to ALL children
② AuthContext.jsx — Global Auth State Lives in memory for entire session. Any component can read it.
State managed here
useState — user useState — loading useEffect — on mount
user = the logged-in user object (or null)
loading = true while checking token on page refresh

useEffect on mount:
Reads token from localStorage → calls /api/auth/me → sets user state.
This is how the app knows you're still logged in after refresh.
Functions exposed via Context
login(email, pass)
→ POST /auth/login → saves token → setUser()

register(username, email, pass)
→ POST /auth/register → saves token → setUser()

logout()
→ removes token from localStorage → setUser(null)

updateUser(updatedUser)
→ directly sets user state (used after profile edit)
useAuth() hook — how components use it
Any component calls:
const { user, login, logout } = useAuth()

Used in:
AppRoutes → check if logged in for routing
Layout → show username / avatar / logout
ProfilePage → read + update user info
LoginPage → call login()
RegisterPage → call register()
Router reads auth state → decides which component renders for each URL
③ Routing — AppRoutes in App.jsx React Router v6 · URL controls what renders
URL Path Guard Component rendered What it does If guard fails → redirect to
/login PublicRoute LoginPage Email + password form. Calls login() from AuthContext / (already logged in)
/register PublicRoute RegisterPage Username + email + password. Calls register() / (already logged in)
/ (index) PrivateRoute Layout → DashboardPage Main note grid. Search, filter, create/edit notes /login (not logged in)
/archive PrivateRoute Layout → ArchivePage Shows archived notes. Unarchive or delete /login
/profile PrivateRoute Layout → ProfilePage Edit username, email, upload avatar /login
PublicRoute logic
Uses useAuth() → reads user and loading
If loading → show spinner (prevents flash)
If user exists → <Navigate to="/"> (kick out)
If no user → render children (show the page)
PrivateRoute logic
Uses useAuth() → reads user and loading
If loading → show spinner (token check in progress)
If no user → <Navigate to="/login">
If user exists → render children (allow in)
loading state — why it matters
On page refresh, useEffect in AuthContext runs async.
Without loading guard, React would flash
/login for a split second even if token is valid.
loading=true shows a spinner until the
/auth/me check completes.
PrivateRoute renders Layout → Layout renders the page via <Outlet>
④ Layout.jsx — Persistent Shell Renders once. Sidebar stays. Only the <Outlet> slot changes per route.
Hooks used
useAuth() useState — tags[] useState — showTagsModal useState — selectedTag useEffect — fetchTags useNavigate() useToast()
tags[] = list of user's tags shown in sidebar
showTagsModal = open/close TagsModal
selectedTag = which tag is active filter
useEffect = fetch tags from API once on mount
What Layout renders
<div class="app-layout"> <Sidebar> ← always visible, same across all pages Logo, nav links (Dashboard/Archive/Profile) Tag list (from tags[] state) User avatar + name (from useAuth().user) Logout button → logout() → navigate('/login') <main> <Outlet /> ← THIS is where DashboardPage/ArchivePage/ProfilePage renders <TagsModal /> ← only if showTagsModal = true <ToastContainer /> ← always present, shows notifications
useOutletContext — how Layout passes data DOWN to pages
Layout passes {'{ tags, fetchTags, selectedTag, setSelectedTag, toast }'} via <Outlet context={...}>
Each page calls const {'{ tags, fetchTags, selectedTag, toast }'} = useOutletContext() to receive it.

This means: DashboardPage knows which tag is selected in the sidebar without prop drilling.
Outlet slot fills with the matched page component
⑤ Pages — What renders inside <Outlet> Each page is an independent component. All receive context from Layout via useOutletContext.
DashboardPage.jsx — route: /
Hooks used
useState — notes[] useState — loading useState — search useState — typeFilter useState — showModal useState — editNote useEffect — fetchNotes useCallback — fetchNotes useOutletContext()
What each hook does
notes[] = array of note objects from API
search = text in search box. useEffect re-runs when it changes (300ms debounce)
typeFilter = 'all' | 'text' | 'image' | 'link' | 'file' — filters the API query
showModal = controls whether NoteModal is visible
editNote = null (create mode) OR a note object (edit mode) passed to NoteModal
useCallback(fetchNotes) = memoises the fetch function so useEffect doesn't loop
useEffect = watches [search, typeFilter, selectedTag] → calls fetchNotes with params
useOutletContext = gets tags, fetchTags, selectedTag, toast from Layout
Renders
Topbar (search box + New Note btn) FilterBar (type buttons) notes-grid if pinned: section + NoteCards if unpinned: NoteCards if empty: EmptyState if showModal: NoteModal
ArchivePage.jsx — route: /archive
useState — notes[] useState — loading useEffect — fetchArchived useOutletContext()
Simpler page. useEffect fires once on mount → GET /notes/archived → fills notes[].
No search, no filter. Just renders archived NoteCards with Unarchive + Delete actions.
Topbar (title only) notes-grid NoteCard × archived notes if empty: EmptyState
ProfilePage.jsx — route: /profile
useAuth() useState — form {username, email} useState — loading useState — avatarLoading useOutletContext()
No useEffect needed — user data comes from useAuth().user directly (already in memory).
form is initialised from user object.
Avatar upload → PUT /upload/avatar → calls updateUser() to refresh context.
Topbar profile-grid Avatar card (click → file input → upload) Profile form (username + email + Save)
LoginPage.jsx — route: /login
useAuth() useState — form useState — loading useState — error useState — showPass useNavigate()
Submit → login(email, pass) from AuthContext
On success → navigate('/')
On fail → setError(message) → show inline error
RegisterPage.jsx — route: /register
useAuth() useState — form useState — loading useState — error useState — showPass useNavigate()
Submit → register(username, email, pass)
On success → navigate('/')

Props: toasts[] from useToast()

No hooks. Pure render.
Maps toasts array → renders floating

useToast custom hook — shared across Layout + all pages
⑦ Custom Hooks Reusable logic extracted out of components
useToast.js
useState — toasts[] useCallback — addToast
toasts[] = array of {id, message, type}

addToast(message, type, duration):
1. Generates unique id
2. Adds toast to array → ToastContainer renders it
3. setTimeout → removes it after duration (3s)

Exposes shorthand object:
toast.success(msg) toast.error(msg)
toast.warning(msg) toast.info(msg)

Used in Layout → passed via Outlet context to all pages.
How toast flows through the app
Layout.jsx → const {'{ toasts, toast }'} = useToast() ↓ passes toast into <Outlet context={'{ toast }'}> ↓ every page gets it via useOutletContext() ↓ pages pass it as prop to NoteModal, TagsModal → they call toast.success() or toast.error() ↓ toasts[] state updates in Layout → ToastContainer re-renders → notification appears
api.js — all HTTP calls go through here
⑧ api.js — Fetch Utility (not a hook, not a component) Plain JS module. Every page/component imports it directly.
What it does
Single request() function wraps native fetch.

Auto-reads JWT from localStorage.getItem('braincache_token')
→ adds Authorization: Bearer token header automatically.

Detects FormData → skips Content-Type (browser sets multipart boundary).
Throws errors with the server's message so components can catch them.
Usage pattern everywhere
const data = await api.get('/notes')
const data = await api.post('/notes', formData)
await api.delete('/notes/5')

Always called inside try/catch:
catch(err) → toast.error(err.message)

VITE_API_URL env var sets the base URL
(http://localhost:5000/api in dev)
⑨ Initial Render Flow — What happens when you open the app
First Load (not logged in)
1
Browser loads index.html → runs main.jsx
ReactDOM.createRoot().render(<App/>)
2
App renders AuthProvider → initial state: user=null, loading=true
3
AuthProvider's useEffect runs → checks localStorage for token
No token found → sets loading=false, user stays null
4
AppRoutes renders → calls useAuth() → user=null, loading=false
5
URL is "/" → PrivateRoute → no user → <Navigate to="/login">
6
LoginPage renders — form is empty, waiting for user input
7
User submits → login() → API returns token + user
Token saved to localStorage, setUser(user) called
8
AuthContext updates → user is now set → React re-renders
PublicRoute sees user → redirects to "/"
9
Layout mounts → useEffect fetches tags → Sidebar + Outlet render
DashboardPage mounts → useEffect fetches notes → grid renders
Page Refresh (already logged in)
1
main.jsx runs again → App renders → AuthProvider mounts
user=null, loading=true initially
2
AuthProvider useEffect → finds token in localStorage
Calls GET /auth/me with the token
3
While waiting: loading=true → PrivateRoute shows spinner
No flash to /login — spinner blocks premature redirect
4
/auth/me responds → setUser(user), setLoading(false)
5
PrivateRoute re-renders → user exists → renders Layout + page
App looks exactly as before the refresh
useState local component state
useEffect side effects / API calls
useAuth() reads AuthContext
useCallback memoised function
useNavigate() programmatic routing
useOutletContext() data from Layout
